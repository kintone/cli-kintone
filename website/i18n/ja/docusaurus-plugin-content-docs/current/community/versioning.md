# バージョニングポリシー

このページでは、cli-kintone のバージョン番号をどのように更新するかを説明します。

:::info

cli-kintone の安定性管理方法については、まず[安定性指標](./stability-index.md)を参照してください。

:::

## 基本

バージョニングは[セマンティック バージョニング](https://semver.org/)を参考にしています。

バージョン番号の形式は **x.y.z** です。

## パッチアップデート

**バグ修正または雑務**をリリースする場合、**パッチアップデート**です。**z** の番号を増やします。

例: バグ修正、依存関係の更新、ドキュメントの変更など

対応するコミットタイプは、
`fix`、`test`、`build`、`ci`、`docs`、`perf`、`refactor`、`revert`、`lint`、`style`、`debug`、および `chore` です。

## マイナーアップデート

**新機能**をリリースする場合、**マイナーアップデート**です。**y** の番号を増やします。

例: 新しいコマンドの追加、新しいフォーマットのサポートなど

対応するコミットタイプは `feat` です。

## メジャーアップデート

**破壊的変更**をリリースする場合、**メジャーアップデート**です。**x** の番号を増やします。

例: 出力フォーマットの変更、プラットフォームサポートの終了など

対応するコミットタイプは、`!` フラグが付いた任意のタイプです。
