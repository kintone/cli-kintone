# record delete

`delete`コマンドは、指定されたKintoneアプリのレコードを削除します。

**注意**

- このコマンドはAPIトークン認証のみをサポートしています。
- この操作はロールバックできません。

## 例

```shell
cli-kintone record delete \
  --base-url https://${yourDomain} \
  --api-token ${apiToken} \
  --app ${kintoneAppId} \
  --file-path ${filepath}
```

`--yes`または`-y`オプションを使用して、確認ステップをバイパスできます。

## オプション

共通オプションについては、[オプション](/guide/options)ページをご覧ください。

| オプション     | 必須 | 説明                                                                                 |
| -------------- | ---- | ------------------------------------------------------------------------------------ |
| `--app`        | はい | アプリのID                                                                           |
| `--file-path`  |      | ソースファイルのパス。                                                               |
| `--encoding  ` |      | 文字エンコーディング<br/>デフォルト：`utf8`<br/>エンコーディング：`utf8`および`sjis` |
| `--yes`、`-y`  |      | レコードを強制的に削除                                                               |

## すべてのレコードを削除

`--file-path`オプションが指定されていない場合、対象アプリのすべてのレコードが削除されます。

## 特定のレコードを削除

`--file-path`オプションを指定することで、特定のレコードを削除できます。

`--file-path`の値はCSVファイルへのパスである必要があり、以下の要件を満たす必要があります：

- ファイル拡張子は「.csv」である必要があります。
- CSVファイルのヘッダー行は、対象アプリで定義されているレコード番号フィールドコードである必要があります。
- レコード番号にアプリコードを使用する場合：
  - すべての行に同じアプリコードが含まれている必要があります（混在不可）。
  - アプリコードは対象アプリのものと一致する必要があります。
