# フィールド仕様

基本的に、入力は[APIリクエスト](https://kintone.dev/en/docs/kintone/overview/field-types/)の`value`プロパティと同じである必要があります。

## レコード情報

| フィールドタイプ | 入力             |
| ---------------- | ---------------- |
| レコード番号     | APIリクエストと同じ |
| レコードID       | サポートされていません |
| リビジョン       | サポートされていません |
| 作成者           | APIリクエストと同じ |
| 作成日時         | APIリクエストと同じ |
| 更新者           | APIリクエストと同じ |
| 更新日時         | APIリクエストと同じ |

## 一般フィールド

| フィールドタイプ | 入力                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------ |
| 文字列（1行）    | APIリクエストと同じ                                                                        |
| 文字列（複数行） | APIリクエストと同じ                                                                        |
| リッチエディター | APIリクエストと同じ                                                                        |
| 数値             | APIリクエストと同じ                                                                        |

| チェックボックス | APIリクエストと同じ                                                                        |
| ラジオボタン     | APIリクエストと同じ                                                                        |
| 複数選択         | APIリクエストと同じ                                                                        |
| ドロップダウン   | APIリクエストと同じ                                                                        |
| ユーザー選択     | [ユーザー選択、組織選択、グループ選択](./user-group-org-selection.md)                     |

## cli-kintoneがCSVファイルから読み取るフィールドを選択する方法

importコマンドを実行する場合

- `--fields`オプションなし：
  - Kintoneアプリに存在するCSVファイル内のすべてのフィールドがインポートされます。
- `--fields`オプションあり：
  - [対象フィールドの指定](../target-fields.md)ページを参照してください。

## cli-kintoneがCSV内の欠落フィールド/空文字列を処理する方法

- CSVファイルのヘッダーにフィールドが欠落している場合 => cli-kintoneはそのフィールドを無視します。
- 行のフィールド値が空文字列の場合 => cli-kintoneはそのフィールドを空文字列で追加/更新します。
